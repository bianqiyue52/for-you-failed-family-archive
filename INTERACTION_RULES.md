# Interaction Rules

- Every click damages the archive.
- Repeated clicks do not reveal truth; they create conflicting records.
- Child / parent / grandparent fragments overlap rather than resolve.
- Silence is recorded as an event.
- Watering is care labor inside an administrative grid.
- The final state is not resolution, but classification failure.

## System Logic

The prototype should preserve the existing technical behavior: raycasted object clicks, persistent traces, fragment overlap, the hold-to-fill watering interaction, soil patch watering, and the final black fade.

The meaning of those actions should shift toward archive failure. A mark is not dirt; it is a damaged record. A fragment is not a ghost; it is a misfiled statement. A blackout is not closure; it is a classification system that can no longer display its own categories.
